KARL manual testing

Selenium

To run these selenium suites in interactive mode, you have to symlink (or copy) karl/seleniumtests to karl/views/static/seleniumtests in the karl package, first. Running them does not require Selenium RC (Remote Control), and should work from any supported browser.

  • all_suite.html To run this suite, you need to create the Zope account with bin/adduser jpglenn09 ******** , look for the password in the selenium tests.
    On Safari or Chrome, you need to log in as this user manually first, before running the suite. (reason unknown)
  • static-only.html This automates the run of all qunit tests with a single click. No server is needed.

Individual modules (demos and qunit)

The pages marked as "Demo" serve for visual, interactive testing, without a need for a server. They can be run direct from the source repository url, or from the static filesystem.
